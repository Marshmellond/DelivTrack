> 最后更新：2026-06-25

---

## 一、项目概述

外卖配送实时监控与运营管理系统。Python 模拟器生成订单 → Kafka 缓冲 → Flink 实时计算 → MySQL 存储 → FastAPI 接口 → Next.js 前端展示。

### 核心功能
- **实时大屏**：6 指标卡 + 6 图表 + 直播订单流
- **数据分析**：订单/营收/配送/客户共 4 个分析页
- **运营中心**：订单处理台 + 骑手调度中心
- **后台管理**：用户/商家/骑手/菜品/订单/区域增删改查
- **系统监控**：Kafka/Flink/MySQL 实时健康检测
- **用户认证**：JWT 登录注册，admin/user 角色权限

---

## 二、技术架构

```
模拟器(500条/秒) → Kafka(3 Broker) → Flink(5管道) → MySQL(9表)
                                                          ↓
Next.js(18页面) ← FastAPI(40+接口) ←───────────────────┘
```

| 层级 | 技术 | 说明 |
|------|------|------|
| 数据生成 | Python + kafka-python + pymysql | generator/ 独立项目 |
| 消息队列 | Kafka 2.8.2 | 3 Broker，Topic: delivery-orders |
| 流计算 | Flink 1.17.2 (HA) | DataStream API，5 管道 |
| 存储 | MySQL 5.7 | Node_02，delivery_dashboard 库 |
| API | FastAPI + JWT + bcrypt | 40+ REST 端点 + WebSocket |
| 前端 | Next.js 16 + ECharts + TailwindCSS v4 | 18 页面，暗色主题 |

## 四、数据库设计（9 表）

### 基础表
- **users** — 用户（id, username, password, phone, address, role, create_time）
- **merchants** — 商家（id, name, category, city, address, phone, status, create_time）
- **riders** — 骑手（id, name, phone, city, vehicle, status, create_time）
- **menu_items** — 菜品（id, merchant_id, name, price, stock, category, status, create_time）

### 流水表
- **orders** — 订单（id, order_no, user_id, merchant_id, rider_id, items(JSON), total_amount, delivery_fee, distance, status, city, district, create_time, accept_time, delivery_time, finish_time）

### 聚合表
- **dashboard_summary** — 全局摘要（1 行，gmv, order_count, rider_online, rider_delivering, avg_delivery_time, cancel_rate, avg_order_amount）
- **dashboard_region** — 区域统计（city, district, order_count, gmv）
- **dashboard_merchant_rank** — 商家排行（merchant_name, category, order_count, gmv）
- **dashboard_hourly** — 时段趋势（time_slot, order_count, gmv, avg_delivery_time）

完整 SQL 见 `docs/full-schema.sql`。

## 六、Flink 管道

| # | 管道 | 输出 |
|---|------|------|
| 1 | windowAll 5s 全局聚合 | dashboard_summary (REPLACE INTO) |
| 2 | keyBy city+district | dashboard_region (ON DUPLICATE KEY UPDATE) |
| 3 | keyBy merchant_name | dashboard_merchant_rank |
| 4 | 复用摘要流 | dashboard_hourly (INSERT + 清理) |
| 5 | 逐条写入 | orders (INSERT) |

## 九、启动流程

```bash
# 1. 集群
ssh root@192.168.157.121
start-all.sh

# 2. Flink JAR（IDEA Maven clean package 后）
# 浏览器打开 http://192.168.157.121:8081 → Submit JAR

# 3. 种子数据（仅首次）
cd server && uv run python seed_full_data.py

# 4. 本地服务（3 终端）
cd generator && uv run python simulator/run.py
cd server && uv run uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload
cd web && pnpm dev

# 5. 打开
http://localhost:3000
# 管理员：user_0000 / 123456
```
